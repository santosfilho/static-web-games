1. Visão Geral

A Câmara do Eco é um motor de análise de sequências numéricas que detecta automaticamente três tipos de padrões matemáticos e prevê valores futuros:

  • PA — Progressão Aritmética
  • PG — Progressão Geométrica
  • POLI — Sequência Polinomial (grau 2+)

O motor usa diferenças finitas e análise de razões para classificar automaticamente qualquer sequência fornecida, priorizando os padrões mais simples primeiro.

2. Progressão Aritmética (PA)

Uma PA é uma sequência onde a diferença entre termos consecutivos é constante. Chamamos essa diferença de razão (d) ou diferença comum.

FÓRMULA DO TERMO GERAL a(n) = a₁ + (n - 1) × d

Propriedades

  • Diferença constante: a(n+1) - a(n) = d, para todo n
  • Soma dos n primeiros termos: S(n) = n × (a₁ + aₙ) / 2
  • Termo médio: a(k) = (a(k-1) + a(k+1)) / 2

Exemplos no Jogo

SequênciadPróximo
[2, 4, 6, 8]+210
[30, 24, 18, 12]-66
[-8, -5, -2, 1]+34

Detecção

O algoritmo calcula as diferenças finitas de primeira ordem (Δ¹) e verifica se são todas iguais (com tolerância de ε = 10⁻⁹ para ponto flutuante). 

Δ¹ = [a₂-a₁, a₃-a₂, ..., aₙ-aₙ₋₁]
Se todos Δ¹ᵢ ≈ constante → é PA

3. Progressão Geométrica (PG)

Uma PG é uma sequência onde a razão entre termos consecutivos é constante. Chamamos essa razão de q.

FÓRMULA DO TERMO GERAL a(n) = a₁ × q^(n-1)

Propriedades

  • Razão constante: a(n+1) / a(n) = q, para todo n (aₙ ≠ 0)
  • Soma infinita (|q| < 1): S = a₁ / (1 - q)
  • Crescimento: exponencial se |q| > 1, decrescente se |q| < 1

Exemplos no Jogo

SequênciaqPróximo
[3, 6, 12, 24]248
[2, 6, 18, 54]3162
[256, 128, 64, 32]0.516

Detecção

Calcula-se a razão a(i+1)/a(i) para cada par consecutivo. Se todas são iguais (e nenhum termo é zero), é PG. 

razões = [a₂/a₁, a₃/a₂, ..., aₙ/aₙ₋₁]
Se todos razõesᵢ ≈ constante E nenhum aᵢ = 0 → é PG

Restrição Importante

Se a sequência é constante (ex: [5, 5, 5]), ela é classificada como PA com d=0 (prioridade sobre PG com q=1), pois PA é o padrão mais simples.

4. Sequência Polinomial

Uma sequência é polinomial de grau k se existir um polinômio p(n) de grau k tal que p(0) = a₀, p(1) = a₁, ..., p(n) = aₙ.

Equivalentemente: a k-ésima diferença finita é constante (e a (k-1)-ésima não é).

INTERPOLAÇÃO DE NEWTON (Diferenças Avançadas) p(x) = Σ C(x, k) × Δᵏ[0]
onde C(x,k) = x! / (k! × (x-k)!) = coeficiente binomial

Tabela de Diferenças Finitas

Exemplo para a sequência [1, 4, 9, 16, 25] (n²):

Nível 0 (original): 1    4    9   16   25
Nível 1 (Δ¹):         3    5    7    9
Nível 2 (Δ²):            2    2    2        ← constante!
                                               → grau 2

Exemplos no Jogo

SequênciaGrauPadrãoPróximo
[1, 4, 9, 16, 25]2n²36
[1, 3, 6, 10, 15]2n(n+1)/221
[1, 8, 27, 64, 125]3n³216

Previsão

Uma vez determinados os coeficientes de Newton Δ⁰[0], Δ¹[0], Δ²[0], ..., Δᵏ[0], o próximo valor é calculado avaliando p(n) com x = tamanho da sequência original: 

p(x) = Δ⁰[0]×C(x,0) + Δ¹[0]×C(x,1) + Δ²[0]×C(x,2) + ... + Δᵏ[0]×C(x,k)

5. Algoritmo de Detecção

O motor analisa a sequência na seguinte ordem de prioridade:

  1. PA — Verifica se Δ¹ é constante (O(n))
  2. PG — Verifica se as razões são constantes (O(n))
  3. Polinomial — Aplica diferenças finitas até grau 10 (O(n×k))
  4. Desconhecida — Nenhum padrão reconhecido

A prioridade garante que padrões simples nunca sejam "sobre-classificados". Por exemplo, toda PA é tecnicamente um polinômio de grau 1, mas é reportada como PA por ser a explicação mais simples (princípio da parcimônia / Navalha de Occam).

PSEUDOCÓDIGO analisarSequencia(seq):   se diferencasFinitas(seq) todas iguais → retorna PA   se razões(seq) todas iguais → retorna PG   para grau = 2..10:     se Δᵍʳᵃᵘ(seq) todas iguais → retorna POLI(grau)   retorna DESCONHECIDA

6. Otimização de Desempenho

Complexidade

OperaçãoComplexidadeObservação
Detecção PAO(n)Uma passagem linear
Detecção PGO(n)Uma passagem linear
Detecção PolinomialO(n × k)k = grau máximo (10)
Previsão (PA/PG)O(1)Fórmula direta
Previsão (Poli)O(k)Avaliação binomial

Estratégias Aplicadas

  • Early return: para no primeiro padrão encontrado (PA > PG > Poli)
  • Pré-alocação de arrays: evita push em laços críticos
  • Cache de fatorial: evita recalcular valores já computados
  • Tolerância ε relativa: comparação robusta para ponto flutuante
  • Limite de grau: máximo 10 para evitar explosão em sequências ruins

Benchmarks

PA com 10.000 elementos   → detectada em ~2ms
PG com 1.000 elementos    → detectada em ~1ms
Poli grau 2 (500 elem.)   → detectada em ~3ms

7. API do Motor (engine.js)

O motor exporta um objeto EchoChamberEngine (global no browser, module.exports no Node.js).

Funções Principais

FunçãoDescrição
analisarSequencia(seq) Retorna análise completa: tipo, fórmula, parâmetros
preverProximos(seq, n) Prevê os próximos n valores (padrão: 1)
gerarSequencia(analise, comprimento) Gera sequência a partir da análise detectada
registrarNoHistorico(seq, analise, previsao) Adiciona ao histórico global
estatisticasHistorico() Retorna estatísticas agregadas

Exemplo de Uso (Node.js)

const Engine = require('./engine.js');

const resultado = Engine.preverProximos([1, 4, 9, 16, 25]);
console.log(resultado.valores);    // [36]
console.log(resultado.analise.tipo); // 'polinomial'
console.log(resultado.analise.grau); // 2

Exemplo de Uso (Browser)

<script src="engine.js"></script>
<script>
  const r = EchoChamberEngine.preverProximos([2, 6, 18, 54]);
  alert('Próximo: ' + r.valores[0]); // 162
</script>

8. Suíte de Testes

A suíte de testes (tests.js) cobre 6 categorias com 40+ asserções:

  1. PA: básica, negativa, decimal, constante, dois elementos, múltiplas previsões
  2. PG: básica, fracionária, negativa, prioridade PA vs PG
  3. Polinomial: quadrática, cúbica, triangulares, previsão múltipla
  4. Validação: array vazio, NaN, Infinity, tipo errado
  5. Utilitários: binomial, fatorial, diferenças finitas, aproxIgual
  6. Desempenho: PA 10k, PG 1k, Poli 500 — todas < 500ms

Executar Testes

cd echo-chamber
node tests.js
